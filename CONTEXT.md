# CuteSkin — Тест на вакансию Project Manager

Полный контекст проекта. Документ для последующих сессий — содержит всё, что нужно, чтобы продолжить работу с нуля.

---

## 1. Назначение

Одностраничный сайт с тестом для собеседования кандидатов на позицию **Project Manager** в бренд женской косметики **CuteSkin** (продажи на Wildberries / Ozon).

Тест проверяет:
- знание терминов (маржа, ДРР, CPO, границы трафика)
- понимание воронки продаж
- умение находить причины изменений метрик (падение заказов, конверсии и т.п.)
- математические расчёты по воронке

## 3. Структура теста

**4 раздела × 3 вопроса = 12 вопросов.** Порядок строго зафиксирован.

### Раздел 1. Термины (single-choice, 1 правильный)
1. Что такое маржинальная прибыль? — *правильно: «Разница между выручкой и всеми переменными расходами…»*
2. Что такое границы трафика? — *правильно: «Позиция ниже которой находится не более 20% трафика»*
3. Что такое СРО? — *правильно: «Сумма затрат на рекламу / количество заказов»*

### Раздел 2. Факторы влияния (multi-choice, несколько правильных)
4. Что влияет на CTR в поисковой выдаче? — правильны ответы 2, 3, 5
5. Что может резко снизить место в выдаче? — правильны ответы 1, 2, 4
6. Что влияет на трафик в рекомендательных полках? — правильны ответы 1, 3, 5

### Раздел 3. Ситуации (single, 1 правильный)
7. Упало количество заказов при неизменной конверсии — *уменьшилось количество показов карточки*
8. Снизили цену, но маржа выросла — *увеличились конверсии → СРО снизился*
9. Упала конверсия в корзину с 20% до 14% — *товар добавили в склейку*

### Раздел 4. Математические (single, 1 правильный)
10. Цена 1000→800, СРО стал 50 руб. — *ДРР стал 6%*
11. Два товара с одинаковой ценой и трафиком, разные конверсии и ДРР — *заработок одинаковый*
12. CR вырос 10→20%, выручка упала на 20% — *CTR снизился с 5% до 2%*

Все варианты ответов с точными формулировками — в массиве `QUIZ` в `index.html`.

## 4. Логика поведения

### Single-choice (разделы 1, 3, 4)
- Один клик финализирует вопрос.
- Подсвечивается **только кликнутый** ответ: зелёный если правильно, красный если нет.
- Остальные варианты становятся недоступными, но нейтральными (не раскрываем «правильный» сами).

### Multi-choice (раздел 2, поле `multi: true`)
- Каждый клик независимо подсвечивает **только кликнутый** ответ (зелёный/красный).
- Остальные варианты остаются кликабельными — пользователь может выбрать несколько.
- `pick()` добавляет индекс в `s.picked[]`, не блокирует остальные.

### Общее
- **Изменить уже кликнутый ответ нельзя** (`1 раз нажал и всё»).
- Переход между вопросами — в любой момент: кнопки «← Назад» / «Дальше →» или клик по номеру в шапке.
- На 12-м вопросе — кнопка «Завершить тест» → экран с общим счётом и разбивкой по разделам.
- Кнопки «Пересмотреть ответы» и «Пройти заново» на финальном экране.

## 5. Дизайн

### Шрифты (строгие, не декоративные)
- **Inter** (400–800) — основной текст, подписи, кнопки
- **Playfair Display** (600–700) — заголовок H1, текст вопросов, цифры результата
- Подключены через Google Fonts
- ❌ Отвергнут Cormorant Garamond (слишком декоративный)

### Палитра
| Назначение | Цвет |
|------------|------|
| Основной текст | `#111111` / `#000000` (чёрный) |
| Акцент бренда (primary) | `#d9466f`, `#ec7aa0` |
| Светлый розовый (backgrounds) | `#fcd4e0`, `#f5a5c0`, `#fbc4d3` |
| Зелёный (правильный ответ) | `#7fb884 → #4a8650` |
| Красный (неверный ответ) | `#d85464 → #b22d45` |

### Фон
Многослойный насыщенный розовый:
```css
radial-gradient(circle at 10% 5%, #fbc4d3 0%, transparent 50%),
radial-gradient(circle at 90% 95%, #f8a8c0 0%, transparent 50%),
radial-gradient(circle at 50% 50%, #fcd4e0 0%, transparent 70%),
linear-gradient(135deg, #fddce6 0%, #f9bdd0 60%, #f5a5c0 100%)
```

### Компоненты
- **Шапка и карточки**: стекло-эффект — `rgba(255,250,252,0.92)` + `backdrop-filter: blur(14px)` + мягкие розовые тени, `border-radius: 28px`
- **Кнопки и варианты**: «пузырчатые» 3D — linear-gradients + многослойные box-shadow (внешняя тень + inset highlight сверху + inset down-shadow снизу), скругление 22–999px, hover-подъём `translateY(-2px)`
- **Логотип**: `<img src="logo.png" class="logo">` по центру шапки, `width: 160px`, `filter: drop-shadow(0 6px 14px rgba(217,70,111,0.25))`
- **Навигация**: сетка из 12 круглых кнопок-таблеток. Активная — малиновый градиент, отвеченные — бледно-розовые
- **Прогресс-бар**: розово-малиновый градиент с glow

### Типографика
- Body: 17px / weight 500
- H1: 38px Playfair Display / 700
- Вопрос: 26px Playfair Display / 700
- Ответ: 16px Inter / 500
- Маркеры А/Б/В/Г/Д: 34×34 круги, 15px / 700

## 6. Запуск

### Локально
```bash
cd "c:/Users/user/Desktop/Клауд/Папка для теста"
python -m http.server 8000
```
Открыть: `http://localhost:8000/`

### Публичная ссылка (SSH-туннель)
Используем **localhost.run** — чистый HTTPS, без заглушек, без аккаунта:

```bash
ssh -o StrictHostKeyChecking=accept-new -R 80:localhost:8000 nokey@localhost.run
```

Выдаёт URL вида `https://<hash>.lhr.life`. Живёт пока запущен сервер + туннель.

### ❌ Что не использовать
- **localtunnel / loca.lt** — показывает страницу «Click to Continue», пользователь просит избегать
- **ngrok** — требует аккаунт

### Постоянный хостинг (ещё не сделан)
Варианты: Netlify Drop, Vercel, GitHub Pages, Cloudflare Pages. Все требуют регистрацию. Если понадобится — провести пользователя через создание аккаунта.

## 7. История итераций в этой сессии

1. **Первый запуск** — тёмно-синие (navy) акценты → отвергнут, запрошена косметическая палитра
2. **Multi-вопросы** изначально финализировали все варианты и раскрывали все правильные → переделано: каждый клик показывает только себя
3. **Шрифты**: Cormorant Garamond + градиентный H1 → заменены на Inter + Playfair Display + чёрный текст
4. **Фон**: пастельно-бежевый → запрошен «более розовый», насыщенность усилена
5. **Логотип**: добавлен `logo.png` (вставлен из clipboard) в центр шапки
6. **Размер текста**: несколько раз увеличен — жалоба «плохо видно»

## 8. Что НЕ трогать / подводные камни

- **Порядок вопросов** фиксированный — менять нельзя
- **Флаг `multi: true`** — только у вопросов 4, 5, 6 (раздел «Факторы влияния»)
- **Логика multi**: клик НЕ должен раскрывать все правильные — только кликнутый
- **Логика single**: клик НЕ должен раскрывать правильный ответ автоматически — только кликнутый (зелёный или красный)
- **Шрифты строгие** — не возвращать декоративные засечки типа Cormorant
- **Текст — чёрный**, не цветной / не градиентный
